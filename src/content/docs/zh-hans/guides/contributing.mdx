## 代码
如果您希望为 AQ 提交代码。您可以先阅读开发者文档的风格指南。然后根据开发者文档的其它文档编写相关代码。

1. Fork [AQ 的 Github 仓库](https://github.com/aq-org/AQ)。（如果您是第一次为 AQ 提交代码）
2. 编写相关代码至 fork 后的仓库。
3. 提交 Pull Request，并附上编写代码的描述。
4. 等待同步代码。（如果我们对您提交的代码有疑问，可能会联系您）

> [!NOTE]
> 如果您是核心开发者（拥有直接向核心存储库提交的权限）：
> 1. 不要直接向核心存储库提交代码。
> 2. 尽量仔细地编写代码（如果您选择直接合并 Pull Request）
> 3. 提交 Pull Request 时，请加上部分描述，便于他人理解。

## Pull Request
### 提交 Pull Request
如果您需要提交 Pull Request，请添加简短的标题或描述说明提交的内容。

### 审查 Pull Request
如果您是核心开发者（拥有直接向核心存储库提交的权限），您可以审查 AQ 仓库的 Pull Request。如果 Pull Request 符合要求，您可以直接合并它。合并 Pull Request 时，请在 commit 的消息写提交内容的描述。不要直接使用 Github 默认的消息。

> [!WARNING]
> 如果对提交内容没有把握，不要直接合并 Pull Request，请先等待其它核心开发者的审查。(无论这个 Pull Request 来自自己还是他人)

## 文档
### 编写文档
如果您希望为 AQ 编写文档，可以根据以下步骤：

1. Fork [AQ Docs 的 Github 仓库](https://github.com/aq-org/docs)。（如果您是第一次为 AQ Docs 提交文档）
2. 编写相关文档至 fork 后的仓库。
3. 提交 Pull Request，并附上编写文档的描述。
4. 等待同步文档。（如果我们对您提交的文档有疑问，可能会联系您）

> [!NOTE]
> 如果您希望为 AQ 编写文档，编写的文档尽量包含英语。

### 翻译文档
翻译文档参照编写文档的步骤。翻译文档时，尽量保持原文档的风格和格式。可以使用相关的工具，但请保证语言的准确性。

如果您有更多的语言需求，可以提出 Issue。如果您会更多语言的翻译，您可以直接添加语言。（添加语言需要在 astro.config.mjs 文件中先添加侧边栏的翻译）

## 报告 Bug
如果您发现了软件、文档或其它作品中的错误、漏洞、缺陷等，您可以在对应的存储库提出 Issue 或与我们联系。我们会尽快处理相关错误。

详细内容见报告 Bug 页面。

## 其它
如果您还有其它问题未能解决，可以直接在 Github 提出 Issue 或与我们联系。如果您有意见或建议，也可以提出 Issue 或联系我们。
